# Catalog website

Static Next.js site that browses the catalog dumps in `../data/`.

## Local development

```bash
# 1. Generate the catalog (one-time, slow)
cd ..
python -m scripts.dump_animewitcher --limit 50
python -m scripts.dump_asia2tv --limit 50

# 2. Run the dev server
cd web
npm install
npm run dev
```

Open <http://localhost:3000>.

## Static build

```bash
npm run build
# output goes to ./out/
```

The `out/` folder is a fully static site — drop it on Vercel,
Cloudflare Pages, GitHub Pages, S3, or any static host.

## Refresh

The catalog is regenerated by the `scrape-and-deploy` GitHub Action
on a nightly schedule (or manually via "Run workflow"). Each run
re-scrapes both sources and rebuilds the site.

## Limitations

- Per-episode server URLs (Pixeldrain, Bunny CDN, …) are **not**
  resolved here; the static dump only links to source episode pages.
  Resolving them would multiply request volume by ~20× and is left
  to a follow-up.
- Asia2TV catalog walks the WordPress sitemaps and filters out
  episode pages by URL heuristic; some series may be missed if their
  slug happens to match the heuristic.
